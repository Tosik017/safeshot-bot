# SafeShot Bot (NoEscapeScreenLink)

Telegram-бот безопасного предпросмотра ссылок для группы. Получив ссылку, открывает её в изолированном headless-Chromium, снимает **первый экран** страницы и отвечает **одним сообщением**: карточка с метаданными (название, цена, бренд, описание) + скриншот + антифишинг-предупреждение. Участники видят, что за ссылкой, не переходя по ней.

## Как это выглядит в чате

1. Ссылка от участника → бот мгновенно отвечает компактной заглушкой-предупреждением (780×320).
2. Через 10–90 с **то же сообщение** превращается (editMessageMedia) в результат:
   - **успех** — скриншот первого экрана (585×960) + карточка с метаданными;
   - **скриншот не вышел, но данные есть** — мини-баннер «скриншот недоступен» + карточка;
   - **полный фейл** — мини-баннер «превью не удалось».
3. Ничего не удаляется, новые сообщения не плодятся — одна ссылка, одно сообщение. Сообщение в ленте растёт только под реальный контент.
4. Доверенные домены (whitelist) — тихая реакция 👌 без сообщений.

## Возможности

- **Полная мобильная эмуляция**: мобильный UA + is_mobile/has_touch + viewport 390×844, DSF 1.5 — сайты отдают настоящую мобильную вёрстку, многие анти-боты пропускают.
- **Метаданные с двух сторон**: httpx-fetch (OG/JSON-LD, свой набор UA) параллельно с браузерным парсингом; берётся лучшее из двух.
- **SSRF-защита**: проверка `security.is_safe()` на входе, повторно перед goto и на **каждый** под-запрос страницы в route handler (subresource/iframe/fetch/redirect на внутренние адреса режутся).
- **Резка лишнего**: media/websocket/fonts + рекламно-трекерные домены блокируются — быстрее и дешевле по памяти; картинки и CSS не трогаются (превью визуальное).
- **Анти-спам**: эскалация на повторы одной ссылки (предупреждение → реальный mute через restrictChatMember), rate-limit разных ссылок, per-chat квота очереди. Админы — мимо всех ограничений.
- **Привязка к группам**: бот сам покидает чужие чаты (allow-list), отдельные топики можно отключить.
- **Кэш**: file_id готовых скриншотов в RAM с TTL — повторная ссылка отвечается мгновенно и без рендера.

## Самозащита по памяти (Render Free, 512 МБ)

Метрика — cgroup (`memory.current − (file − shmem)`, модуль `ram.py`): ровно то, что видит OOM-киллер; сумма RSS не годится (двойной счёт shared-страниц Chromium).

| Порог | Действие |
|---|---|
| 360 МБ | рестарт браузера ПЕРЕД задачей (сброс V8 heap) |
| 430 МБ | только лог `accept under pressure` на входе в очередь |
| 470 МБ | RAM-guard рвёт контекст ТЕКУЩЕЙ страницы во время рендера → текст-фолбэк, рестарт браузера; инстанс живёт |
| 512 МБ | OOM Render — досюда доходить не должно |

Плюс плановый рестарт браузера каждые 50 запросов и аварийное самовосстановление, если браузер отвалился.

## Стек

Python 3.12 · aiogram 3 (polling) · Playwright/Chromium (образ `mcr.microsoft.com/playwright/python:v1.60.0-noble`) · Pillow · httpx · uvicorn (эндпоинты `/`, `/ping`, `/health`) · Docker, non-root (`pwuser`).

## Деплой (Render Free)

- Пуш в `main` → автодеплой. **Внимание:** при упавшем деплое Render продолжает крутить предыдущую версию — «бот отвечает» ≠ «новый код доехал».
- Shell на Free недоступен; форс-рестарт: `git commit --allow-empty -m "restart" && git push` или Suspend/Resume.
- Free засыпает без внешнего HTTP — нужен внешний пингер на `/ping`.
- Health-check: `GET /health` → `{"status":"ok","browser":true,"bot":true,"worker":true}`. Задайте `HEALTH_TOKEN`, чтобы закрыть его заголовком (`curl -H "X-Health-Token: ..." .../health`) — без токена в env эндпоинт остаётся открытым.

## Конфигурация

Обязательное окружение: `BOT_TOKEN`. Опциональное: `TRUSTED_DOMAINS` (дефолт: youtube.com, youtu.be, wikipedia.org, github.com; матч по границе точки, IDN-гомогли­фы не проходят), `CHROMIUM_SANDBOX=on` (для хостов с user namespaces; на Render Free недоступно — компенсируется non-root), `JITLESS=on` (режет RCE-поверхность V8 ценой скорости — по умолчанию выключено). Лимиты очереди/рейтов/памяти — константы в `config.py` с комментариями-причинами.

## Кастомизация графики

Положи свои PNG 780×320 в корень репозитория — бот подхватит по имени файла: `placeholder.png` (заглушка-старт), `banner_text.png` (скриншот недоступен), `banner_fail.png` (превью не удалось). Файлов нет — бот рисует фолбэк-версии сам (Pillow, DejaVu). Смена любого файла = деплой (графика зашивается в образ).

## Известные ограничения

- Один воркер: при всплеске хвост очереди ждёт 60–100 с (карточка показывает позицию в очереди).
- Жёсткие анти-боты (Cloudflare-челлендж) могут попасть в кадр или увести в текст-фолбэк — это предел подхода без платных прокси.
- Мёртвая ссылка скриншотится как есть («Сторінку не знайдено») — это и есть честное превью.
- Состояние (кэш, очередь) живёт в RAM — теряется при редеплое.
